================================================================================
                    ELEVEN ELEVEN EVENTS - PROJECT ARCHITECTURE
                       Balloon Artistry Event Booking Website
================================================================================

PROJECT OVERVIEW:
The Eleven Eleven Events website is a professional balloon artistry booking 
platform built with HTML, CSS, and JavaScript. It features event booking, 
gallery showcase, customer reviews, and email communication.

================================================================================
DATA FLOW & CONNECTIONS
================================================================================

USER INTERACTION FLOW:
────────────────────

1. User opens main.html
   ↓
2. HTML loads CSS files for styling
   ↓
3. HTML loads main.js for interactivity
   ↓
4. User navigates sections (About → Gallery → Reviews → Contact)
   ↓
5. User fills booking form
   ↓
6. main.js validates form data (validation-styles.css shows feedback)
   ↓
7. Form submission → send-email.js (Netlify Function)
   ↓
8. send-email.js uses @emailjs/nodejs to send email
   ↓
9. User receives confirmation

FAQ PAGE FLOW:
──────────────
main.html → (Link) → FAQ.html → (Link back) → main.html

================================================================================
FILE RESPONSIBILITIES
================================================================================

main.html:
- Entry point for the website
- Contains all HTML structure and layout
- Links all CSS and JS resources
- Includes navigation, hero section, gallery, reviews, booking form, footer

FAQ.html:
- FAQ section (frequently asked questions)
- May include FAQ-specific styling
- Cross-links with main.html

main.js:
- Navigation toggling (mobile menu)
- Form validation and submission
- Gallery navigation (if interactive)
- Scroll effects and animations
- Event listeners for user interactions

send-email.js:
- Receives booking form data
- Authenticates with email service
- Formats and sends confirmation emails
- Error handling and logging

CSS Files:
- main.css: Global styling, typography, layout grid
- components.css: Button, card, form, section styling
- responsive.css: Mobile-first responsive breakpoints
- validation-styles.css: Success/error message styling
- gallery.css: Gallery grid/slideshow layout

Img/ Folder:
- Stores all project images
- Logo, icons, gallery images
- Referenced in HTML via relative paths
